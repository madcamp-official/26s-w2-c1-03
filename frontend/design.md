# trip and end — 디자인 가이드 (design.md)

> 이 문서는 Claude Design 프로젝트 **"앱 메인화면 초기 디자인"**
> (`https://claude.ai/design/p/587e8d9f-cbcb-4964-b45a-546a294faaec`)의
> `앱 메인화면.dc.html` 시안 중 **채택된 6개 화면(`2a`, `3b`, `3c`, `3d`, `4a`, `4b`)**을
> 분석하여 정리한 디자인 언어 명세서다. 프론트엔드(Flutter) 개발 중 디자인 판단이
> 필요할 때 이 문서를 기준으로 삼는다. 원본 시안에 없는 화면을 새로 만들 때도, 아래
> 규칙(컬러 토큰·타이포·컴포넌트 패턴)을 그대로 적용해 시안과 톤이 어긋나지 않게 한다.
>
> **채택 화면 6개 외의 시안(`1a`, `1b`, `2b`, `3a`)은 폐기되었다.** 특히 스케줄 탭은
> 단순 타임라인형(`3a`)이 아니라 **지도+목록형(`4a`)이 최종안**이라는 점에 주의.
> 이 문서의 모든 예시·수치는 채택된 6개 화면에서만 뽑았다.
>
> **[갱신] 홈 화면만 이후 별도 라운드(`design_example.pdf`, 프로젝트 루트)에서
> 다시 정해졌다.** 그 PDF는 자체적으로 `2a`(심플 카드형)를 "채택"으로 표시하지만,
> 팀 결정으로 **실제 구현은 그 PDF의 `1a`(카드 중심형: 알림 벨 + D-day 히어로 +
> "다음엔 여기 어때?" AI 추천 캐러셀 + 지난 여행 기록)를 따른다.** 이 `1a`는 위에서
> "폐기"라고 한 최초 Claude Design 라운드의 `1a`(퀵액션 4그리드 버전)와 **이름만
> 같고 내용이 다른 별개 시안**이니 혼동하지 말 것 — §6 표 참고.

## 1. 디자인이 추구하는 것

- **"AI가 대신 계획해주는 여행"이라는 제품 본질을 화면에서 계속 느끼게 한다.**
  모든 화면에서 AI가 관여한 콘텐츠(추천 장소, 추천 사진, 일정 초안)는
  `✨` 이모지 + 라임색(`#FAFCBD`) 배지/배경으로 시각적으로 분리해서 보여준다.
  사용자가 직접 만든 콘텐츠와 AI가 만든 콘텐츠를 한눈에 구분할 수 있어야 한다.
- **"토스(Toss) 계열" 한국 핀테크/라이프스타일 앱 문법을 따르는 플랫 미니멀.**
  뉴모피즘이 아니다. 그림자는 CTA 버튼·FAB·플로팅 카드에만 절제해서 쓰고, 나머지는
  배경색 대비와 얇은 보더(1px, `#F2F4F6`)만으로 구획을 나눈다. 입체감보다 여백과
  타이포 위계로 구조를 표현한다.
- **홈 화면은 "카드형" 하나로 통일한다.** 히어로 D-day 카드(라임 배경, `52px` 굵은 숫자)가
  화면당 하나의 압도적 포인트가 되고, 그 아래는 조용한 리스트(지난 여행)로 받쳐준다.
  타이포 단독형(숫자만 거대하게 띄우는 버전)은 실험으로 그쳤고 채택되지 않았다 — 새 화면도
  "라임 카드 + 큰 숫자 + 반말 멘트 + 버튼" 구성을 그대로 따른다.
- **버튼/입력 요소는 각지지 않는다.** 모서리 반경이 전체적으로 매우 크다
  (카드 16~24px, 버튼 14~16px, 칩/배지/탭은 완전한 pill `999px`). 직각에 가까운
  요소가 거의 없다 — 새 컴포넌트를 만들 때도 `border-radius: 4~8px` 같은 값은 이 시스템에
  어울리지 않는다.
- **정보 위계는 컬러 대비가 아니라 명도 대비로 만든다.** 브랜드 컬러(라임)를 텍스트 색으로
  남용하지 않고, 텍스트는 그레이스케일 계층(`#191F28` → `#4E5968` → `#8B95A1` → `#B0B8C1`)
  안에서만 움직인다. 라임/그린 계열은 "AI/브랜드/선택됨" 같은 의미가 있을 때만 등장시킨다.

> 라임(`#FAFCBD`)은 이 디자인의 지문(fingerprint)이다. 그러나 **배경으로만** 쓰고,
> 라임을 텍스트 컬러로 쓰지 않는다 (가독성 문제). 라임 위의 텍스트는 항상 진한 그린 계열
> (`#0B3D2A` / `#0F5132` / `#14532D`)을 쓴다.

### 2.4 프라이머리 버튼 컬러 — 두 종류를 구분해서 쓴다

- **Ink 버튼 (`bg #191F28`, `text #fff`)**: 확정/이동성 액션. 예) "일정 보기", 지도 확장,
  FAB의 배경. 사용자가 이미 결정한 것을 실행할 때.
- **Lime 버튼 (`bg #FAFCBD`, `text #0B3D2A`)**: **AI가 관여하는 생성/추천 액션**.
  예) "✨ AI한테 계획 부탁하기" (`3b`), "✨ 3곳으로 최적 동선 짜기" (`4b`).
  버튼에 AI 개입이 있으면 라임, 없으면 잉크 — 이 구분을 새 화면에도 지킨다.

### 2.5 사진/장소 썸네일 플레이스홀더 그라디언트

## 3. 타이포그래피

- **폰트**: Pretendard (`pretendard.css` CDN) → fallback `-apple-system, system-ui, sans-serif`.
  한글 UI 폰트 표준. 별도 지시 없는 한 다른 폰트로 교체하지 않는다.
- **웨이트는 700(bold)/800(extrabold) 위주.** 400/500 regular는 거의 쓰지 않는다.
  가벼운 정보(캡션, 타임스탬프)도 600 semibold 이상을 쓴다 — 이 앱은 "얇은 글씨"가 없다.
- **자간(letter-spacing)은 D-day 숫자에서만 좁힌다.** 홈 히어로 숫자(`52px`)는
  `-1.5px` negative tracking. 본문/라벨 텍스트는 자간을 건드리지 않는다.

### 3.1 타입 스케일 (실사용 사이즈, 대략 내림차순)

| 크기 | 웨이트 | 용도 |
|---|---|---|
| 52px / 800 | D-day 히어로 숫자 (홈 `2a`, 라임 카드 안) |
| 26px / 800 | 화면 타이틀 — 온보딩형 질문 ("어디로 떠날 거야?", `3b`) |

## 4. 레이아웃 규칙

- **디바이스 기준**: iPhone 프레임, 컨텐츠 뷰포트 `402×874`. 상태바 확보를 위해
  스크롤 콘텐츠 상단 패딩 `62px` 고정. 좌우 패딩은 `20~24px` (보통 `22px`).
- **하단 여백**: 탭바(`~88px`)에 콘텐츠가 가리지 않도록 스크롤 영역 하단 패딩
  `100~120px` 확보. 플로팅 CTA 버튼이 있는 화면(`4b`)은 `120px` 이상.
- **세로 리듬(gap)**: 섹션 간 `18~28px`, 카드 내부 요소 간 `10~16px`, 리스트 행 간은
  gap 없이 `padding: 12~15px 0` + `border-bottom` 구분선으로 처리 (마지막 행은 보더 생략).
- **모서리 반경 스케일**:
  - `12–14px`: 작은 썸네일, day-strip 칩, quick-action 아이콘 박스
  - `16–20px`: 인풋 필드, 지도 컨테이너, 대부분의 카드
  - `24px`: 히어로 카드(D-day 카드) — 화면에서 가장 중요한 요소일수록 반경이 크다
  - `999px` (pill): 배지, 태그, 카테고리 칩, 캡슐형 버튼, 아바타/원형 아이콘
- **그림자는 "떠 있는" 요소에만.** 플로팅 CTA (`0 12px 28px rgba(25,31,40,0.25)`, `4b`),
  FAB (`0 10px 20px rgba(25,31,40,0.3)`), 지도 위 확장 버튼 (`0 2px 8px rgba(0,0,0,0.12)`, `4a`).
  일반 카드/리스트에는 그림자를 넣지 않는다 — 구획은 배경색 대비로만 표현.

## 5. 핵심 컴포넌트 패턴

### 5.1 하단 탭바

5탭 구성 `홈 · 스케줄 · (＋ FAB) · 기록 · 마이`. 배경 `rgba(255,255,255,0.96)` +
`backdrop-filter: blur(10px)`, 높이 `88px`, 상단 보더 `1px solid #EFF1F3`.
중앙 탭은 실제 탭이 아니라 **떠 있는 원형 FAB** (`52px`, `bg #191F28`,
`margin-top: -26px`로 탭바 위로 튀어나옴, 안의 `+` 아이콘은 라임색). 좌우 4개 텍스트
라벨은 `11px`, 활성 탭만 `#191F28` + weight 700, 비활성은 `#B0B8C1` + weight 600.
`3b`(＋ 화면)에서는 이 FAB에 라임색 `outline: 3px solid #FAFCBD`가 추가되어 현재 탭임을 강조한다.

> 참고: 초기 실험(`1b`, `2b`)에는 flat 82px 탭바 + 34px 라임 원형 중앙 버튼 버전이 있었으나
> **채택되지 않았다.** 새 화면도 raised FAB 버전만 사용한다.

### 5.2 D-day 히어로 카드 (`2a`)

홈 화면 최상단, 다가오는 여행이 있을 때 반드시 노출되는 핵심 모듈. 라임 배경 카드
(`24px` 라운드, `padding: 26px`) 하나로 고정.
- 구성 순서 고정: `여행지명 + 날짜 범위` (13~15px/700, `#0F5132`, 상단 좌우 배치) →
  `D-N` (`52px`/800, `#0B3D2A`, `letter-spacing:-1.5px`) → `상황 멘트` (14.5px, `#14532D`,
  반말체 — "여행까지 5일 남았어, 일정 한 번 볼까?") → Ink 버튼 CTA ("일정 보기", 전체 너비).
- 카피 톤은 **반말 + 다정한 챗봇 어조**. "~보자", "~할게", "~됐어?" 같은 구어체를 쓴다.
  존댓말/격식체("확인해 주세요")는 이 화면군에서 쓰지 않는다.

### 5.3 리스트 행 (List Row) — 앱 전체에서 재사용되는 단일 패턴

```
[리딩 요소: 원형 아바타 / 숫자 배지 / 그라디언트 썸네일 / 시간]
  + gap 12~14px +
[타이틀(15px/700, #191F28) 위, 서브텍스트(12.5px, #8B95A1) 아래]
  + (옵션) 트레일링: 체크서클 / chevron(#D1D6DB) / "AI 추천" 배지
padding: 12~15px 0, border-bottom: 1px solid #F2F4F6 (마지막 행 제외)
```
`4a`의 스케줄 리스트, `4b`의 장소 선택 리스트, `3d`의 마이 설정 메뉴가 전부 이 하나의
행 패턴을 변형해서 쓴다 — **새 리스트 화면을 만들 때 이 패턴을 재사용**하고 임의로 새
리스트 스타일을 만들지 않는다.

### 5.4 "AI 추천" 배지 & AI 배너

- 인라인 배지: `background:#FAFCBD; color:#0F5132; font-size:10.5px; font-weight:800; padding:2px 8px; border-radius:999px;` 텍스트: `AI 추천` (`4a` 스케줄 리스트에서 사용).
- 배너형(더 큰 안내, `3c`): 라임 배경 `20px` 라운드 박스, `✨` 이모지 + 진한 그린 2단 텍스트
  (제목 700, 설명 line-height 1.45). 예) "여행이 끝나면 알려줄게 / 사진 중에 베스트 샷
  15장을 골라서 기록을 도와줄게".
- 텍스트 버튼형: `✨` 접두 + 라임 필 배경(`4b`: "✨ 3곳으로 최적 동선 짜기") 또는
  회색 필 배경(`#F2F4F6`) pill 버튼.

**규칙: `✨`가 붙는 모든 요소는 AI가 개입했다는 신호다. 사용자가 직접 만든 콘텐츠에는 `✨`를 붙이지 않는다.**

### 5.5 Day Strip (일자 선택 스트립, `4a`)

가로 스크롤 pill 그룹. 선택됨: `bg #191F28`, 상단 라벨 `#FAFCBD`, 숫자 `#fff`.
미선택: `bg #F2F4F6`, 상단 라벨 `#B0B8C1`, 숫자 `#4E5968`. 반경 `12px` (pill이 아니라 라운드 사각형).

### 5.6 지도 (Map, `4a`)

실제 지도 SDK 스크린샷이 아니라 **손그림풍 flat-illustration SVG**로 표현
(연한 세이지그린 `#E9EDE6` 배경 + 옅은 도로/블록). 장소 마커는 물방울(pin) 모양
(`border-radius: 50% 50% 50% 4px`)의 잉크색(`#191F28`) 배지에 라임색 숫자, 순서가 있는
동선은 마커 번호와 아래 리스트 번호가 1:1로 대응한다. 우측 하단에 흰 원형 확장 버튼.
→ 실제 지도 API를 붙이더라도 **마커 스타일(잉크+라임 숫자, 물방울 핀)과 리스트 연동 패턴은 유지**한다.

### 5.7 선택 인디케이터 (체크서클, `4b`)

장소 선택형 리스트에서 사용: 선택됨 = `26px` 원, `bg #191F28`, 안에 라임색 체크 아이콘.
미선택 = 같은 크기의 outline-only 원 (`border: 1.8px solid #D1D6DB`, 배경 투명).
체크박스/라디오 사각형은 이 디자인에서 쓰지 않는다 — 항상 원형.

### 5.8 카테고리 칩 / 필터 (`4b`)

`bg` 선택됨 `#191F28`+흰 텍스트(700) / 미선택 `#F2F4F6`+`#4E5968`(600), `padding:8px 14px`,
`border-radius:999px`. 가로 스크롤, `overflow-x:auto`.

### 5.9 아이콘

전부 **선(line) 아이콘**, 채우기(fill) 없이 stroke만 사용. stroke-width `1.6~2.2`,
`stroke-linecap: round`, `stroke-linejoin: round` 고정. 커스텀 SVG를 직접 그리는
스타일이며 Material/Cupertino 기본 아이콘 세트와 톤이 다르므로, 아이콘 라이브러리를
쓰더라도 **outline 스타일 + round cap/join**인 세트로 통일한다
(예: Flutter에서는 `CupertinoIcons`보다 outline 계열의 커스텀 SVG나 `phosphor_flutter`(outline)
/ `lucide` 계열이 톤에 더 가깝다).

## 7. 카피 톤 & 보이스

- 반말 + 친근한 챗봇 톤 ("~짜줄게", "~할까?", "~됐어?"). 존댓말 사용 안 함(설정/약관류
  제외 가능성 있으나 채택 시안에는 전부 반말).
- AI 발화는 1인칭으로 제안하듯 말한다: "고른 곳을 전부 넣어서 가장 최적 동선으로 짜줄게" (`4b`),
  "사진 중에 베스트 샷 15장을 골라서 기록을 도와줄게" (`3c`).
- 이모지는 `✨`(AI 관련), `👋`(인사) 정도로 최소한만 사용. 남발하지 않는다.

## 8. 이 디자인 시스템에서 하지 말아야 할 것 (안티패턴)

1. 순수 검정(`#000000`)이나 순수 회색(`#808080` 등 무채색)을 텍스트에 쓰지 않는다 — 반드시
   위 잉크(`#191F28`) 계열 그레이스케일 토큰을 쓴다.
2. 라임(`#FAFCBD`)을 남용해 화면을 노랗게 만들지 않는다. 화면당 라임 사용은
   히어로 카드/배지/1차 CTA 정도로 제한한다.
3. 각진 모서리(4~8px 이하 radius)나 직각 버튼을 쓰지 않는다.
4. 그림자를 일반 카드나 리스트에 남용하지 않는다 — 그림자는 "떠 있는" 요소 전용.
5. 채워진(fill) 아이콘, 두꺼운 duotone 아이콘 등 이 시스템의 line-icon 톤과 다른 아이콘 세트를
   섞어 쓰지 않는다.
6. 탭바 중앙에 일반 탭(아이콘+라벨)을 넣지 않는다 — 반드시 raised FAB (원형, 잉크 배경,
   탭바 위로 튀어나옴) 형태를 유지한다 (§5.1 참고, flat 버전은 폐기됨).
7. 스케줄 탭을 단순 타임라인(`3a`, 폐기)으로 되돌리지 않는다 — 지도+목록 결합형(`4a`)이 최종안이다.
8. AI가 만들지 않은 콘텐츠에 `✨`나 라임 배지를 붙이지 않는다 — 이 신호의 의미가 희석된다.
9. 존댓말 카피를 섞지 않는다 (톤 일관성 붕괴).
